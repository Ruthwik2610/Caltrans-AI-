# Project Architecture Overview: Caltrans Agentic AI Solutions

> [!abstract] Purpose
> This document provides a comprehensive architectural overview of the Caltrans codebase, a Streamlit-based platform hosting multiple specialized AI agents for document intelligence, project evaluation, and LLM operations.

## 🏗️ System Overview

The application follows a modular architecture where a central Streamlit frontend ([[app.py]]) routes requests to specialized modules in the `src/` directory. The system emphasizes **Process-Supervised AI**, where complex evaluations are broken down into verifiable steps with human-in-the-loop feedback mechanisms.

## 🧱 Core Modules

### 1. Application Entry & Routing
- **[[app.py]]**: The main entry point. It manages the sidebar configuration (LLM selection, GCP services) and the main usecase router.
- **[[app2.py]]**: An alternative entry point implementing simple key-based authentication for session control.

### 2. Specialized Evaluators
- **[[src/project_delivery_evaluator.py]]**: 
    - **Function**: Recommends project delivery methods (e.g., Design-Bid-Build, CM/GC).
    - **Logic**: Uses a 25-question rubric extracted from `RUBRIC_QUESTIONS`. It applies weighted scoring across six categories (Scope, Schedule, Innovation, Quality, Cost, Staffing).
    - **Knowledge**: Pulls from an embedded knowledge base in [[src/delivery_method_kb.py]].
- **[[src/cucp_reevals.py]]**:
    - **Function**: Evaluates Social and Economic Disadvantage (SED) narratives.
    - **Multi-Stage Architecture**:
        1. **Level 1 (Fact Extraction)**: Raw extraction of "W5" details.
        2. **Level 2 (Legal Classification)**: Categorizes facts under 49 CFR §26.67.
        3. **Level 3 (Evidentiary Threshold)**: Final Pass/Fail decisions based on a preponderance of evidence.

### 3. Knowledge & Memory Management
- **[[src/memory_manager.py]]**: Implements "Institutional Memory". It captures human overrides during the CUCP evaluation process.
- **[[src/memory_db.json]]**: Stores the persisted human precedents.
- **Consolidation**: Includes logic to use GPT-4o to deduplicate and synthesize overlapping human rules into a clean `rules.json`.

### 4. UI Components
- **[[src/chat_ui.py]]**: A generic chat interface used by multiple usecases (RAG, LLM as a Judge, Training). It handles:
    - Chat history management.
    - Integration with `streamlit_feedback` for thumbs-up/down.
    - Orchestration of specific agent calls (e.g., `policy_agent`, `llm_evaluation_agent`).

### 5. Data & Training
- **[[src/llm_training.py]]**: Handles the training workflow, including data loading from `training_data.jsonl` and performance of model fine-tuning.
- **[[src/llm_evaluation.py]]**: Provides automated evaluation of LLM responses using predefined metrics.

## 🔌 Integration Points
- **[[src/mcp_server.py]]**: Implements a Model Context Protocol (MCP) server, allowing the AI agents to be used as tools or to access expanded context in a standardized way.
- **[[start_mcp.sh]]**: Shell script to initialize and run the MCP server.

## 🛠️ Key Interactions

1. **User Uploads Document**: Handled in [[app.py]] via `st.file_uploader`.
2. **Preprocessing**: Text extraction from PDF/Docx using `PyPDF2` or `python-docx` within the evaluator modules.
3. **Agent Execution**: The relevant evaluator ([[src/cucp_reevals.py]] or [[src/project_delivery_evaluator.py]]) is invoked.
4. **Human Review**: In the CUCP workflow, the user reviews intermediate steps. Corrections are staged and then committed via [[src/memory_manager.py]].
5. **Output Generation**: Results are rendered as Markdown in the UI and can be exported to Excel using `openpyxl`.
