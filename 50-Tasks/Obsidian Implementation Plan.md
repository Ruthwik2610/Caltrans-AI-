## 🛠️ 2. Skill-Specific Workflows

### 🖋️ Obsidian Markdown ([[obsidian-markdown]])
- **Wikilinks**: Always link to source code files (e.g., `[[app.py]]`) and other notes.
- **Callouts**: Use `> [!logic]` or `> [!bug]` to highlight critical sections in documentation.
- **Properties**: Track `status`, `owner`, and `priority` for all major notes.

### 📊 Obsidian Bases
- **Rule Management**: Keep `Institutional Memory.base` aligned with the live precedent store in `src/memory_db.json`.
- **Review Focus**: Store only durable precedents, not raw scratch notes or transient debugging thoughts.

### 🎨 Canvas Notes
- **[[CUCP Evaluation Logic.canvas]]**: A visual map of the 3-level evaluation pipeline.
- **Future canvas**: Add a delivery-method scoring map when the rubric workflow stabilizes.

### 📚 Research Notes
- **Source Library**: Create one note per durable reference, such as `49 CFR Part 26` or an MCP spec summary.
- **Link Discipline**: Research notes should point back to the architecture, task, and memory notes they support.
