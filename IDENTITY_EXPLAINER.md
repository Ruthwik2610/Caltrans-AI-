# How Databricks App Identity Works

When you deploy an application to the Databricks App platform, the system automatically assigns it a **Service Principal Identity**. Instead of requiring you to hardcode a Personal Access Token (PAT) which can expire or be stolen, the platform dynamically injects secure OAuth credentials directly into the application's running environment. By using the `WorkspaceClient().config.authenticate()` method from the Databricks SDK, the code can automatically detect these "ambient" credentials to generate a temporary, auto-refreshing token. This allows the app to securely call Foundation Model APIs and other Databricks services as itself, ensuring that authentication is handled natively by the platform without any manual configuration or secret management.
